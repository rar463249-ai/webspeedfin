Configuration Guide

Getting Started

Step 1: Get Google PageSpeed Insights API Key

  1. Go to the Google Cloud Console
  2. Create a new project or select an existing one
  3. Enable the PageSpeed Insights API
  4. Create credentials (API Key)
  5. Copy your API key

Step 2: Configure the Application

  1. Open analyze.php
  2. Replace YOUR_GOOGLE_PAGESPEED_API_KEY with your actual API key
  3. Save the file

Example:

const API_KEY = 'AIzaSyBOti4mM-6x9WDnZIjIeyEU21OpBXqWBgw';

Step 3: Upload Files

  1. Upload all files to your web server
  2. Ensure PHP is enabled on your server
  3. Make sure the server can make outbound HTTP requests

Step 4: Test the Application

  1. Open index.html in your browser
  2. Enter a website URL to test
  3. Click "Analyze" and wait for results

Server Requirements

  • PHP 7.4 or higher
  • cURL extension enabled
  • JSON extension enabled
  • Outbound HTTP/HTTPS requests allowed
  • Web server (Apache, Nginx, etc.)

Troubleshooting

Error: "Failed to connect to PageSpeed Insights API"

Check that your server can make outbound HTTPS requests and that your API key is correct.

Error: "Invalid URL format"

Make sure the URL includes http:// or https:// and is properly formatted.

Error: "API Error: API key not valid"

Verify your API key is correct and that the PageSpeed Insights API is enabled in your Google Cloud project.
